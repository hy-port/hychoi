Step 01. Requirements 요구사항 파악

"클라이언트가 원하는 것은 무엇인가?"
프로젝트 시작 전 클라이언트의 요구사항을 정확히 파악하는 것이 우선입니다.
디자인 시안과 함께 전달되는 피드백, 참고 사이트, 기능 요청사항을 분석하고
납품 범위와 일정을 명확히 합니다.

에이전시 실무에서 주 3~4건의 프로젝트가 동시에 유입되는 환경에서
각 프로젝트의 우선순위와 복잡도를 빠르게 판단하는 것이 핵심이었습니다.
요구사항이 불명확할 경우 기획자와 사전 커뮤니케이션으로 리스크를 최소화했습니다.
Step 02. Design Analysis 시안 분석 및 구조 설계
"시안을 어떻게 구현할 것인가?"
Figma 시안을 기반으로 마크업 구조와 CSS 레이아웃 방향을 설계합니다.
PC·태블릿·모바일 시안을 비교하여 반응형 분기점을 결정합니다.

모바일 시안이 제공되지 않는 경우,
이때는 메인 시안의 여백·간격·폰트 크기를 기준으로 서브 페이지 반응형을 독립 설계했으며
좌우 패딩·섹션 간격 등 일관된 기준을 수립하여 전체 프로젝트에 반복 적용했습니다.

Step 03. Publishing 퍼블리싱 구현

"시안을 코드로 구현한다."
HTML·CSS·SCSS를 기반으로 시맨틱 마크업과 웹 접근성 기준을 준수하며 퍼블합니다.
인터랙션이 필요한 경우 GSAP ScrollTrigger를 활용하여 스크롤 기반 애니메이션을 구현하고
카카오지도·Chart.js 등 외부 API 연동도 직접 처리합니다.

40건+ 프로젝트를 수행하며 개발 페이지와 퍼블 페이지의
작업 우선순위를 독립적으로 판단하고 일정을 관리했습니다.
크로스브라우징 대응과 반응형 검수까지 납품 전 확인합니다.
Step 04. Maintenance 유지보수 및 운영 대응
"납품 이후도 실무다."
납품 후에도 클라이언트의 수정 요청에 지속적으로 대응합니다.
텍스트·이미지 교체, 페이지 추가, 레이아웃 변경부터
메인 페이지 전체 리뉴얼까지 전 범위를 처리했습니다.

208개사의 유지보수를 담당하며 각 사이트의 구조와 특성을 빠르게 파악하고
CMS 기반으로 콘텐츠를 등록·수정·관리하는 운영 실무를 경험했습니다.

Step 05. QA & Delivery 품질 검수 및 납품

"납품 전 마지막 기준은 내가 잡는다."
납품 전 PC·태블릿·모바일 실기기 및 브라우저별 크로스브라우징을 검수합니다.
퍼블팀 팀장으로서 팀원 작업물의 납품 품질 검수도 담당했습니다.

일정 변동이 발생할 경우 유관 부서 및 기획자와 사전 협의하여
납기를 조율하고 품질을 유지하는 방식으로 대응했습니다.
수행 40건+ 프로젝트에서 납품을 완료했습니다.
Step 06. Retrospective 회고 및 개선
"경험은 기준이 된다."
40건+ 프로젝트를 수행하면서 반복되는 패턴을 발견했습니다.
모바일 시안 미제공, 긴급 요청, 일정 충돌 등 에이전시 특성상 반복되는 상황들을
매번 새로 판단하는 대신 기준을 만들어 표준화했습니다.

이 과정에서 쌓인 판단 기준과 처리 방식이
단순한 퍼블리셔가 아닌 운영 실무자로 성장하는 토대가 되었습니다.
구축과 운영, 두 가지 경험을 바탕으로 다음 단계로 나아갑니다.
